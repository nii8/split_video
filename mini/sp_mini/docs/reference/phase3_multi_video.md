# Phase3: 多视频组合生成

## 目标

第三阶段最小版目标：

- 支持多个视频输入
- 每个视频形成自己的片段池
- 从多个片段池里拼出一个候选
- 给这个候选一个基础兼容评分
- 把兼容评分并回总分

---

## 当前实现

### 最小实现范围

当前第三阶段只做最小骨架：

1. **多视频输入**：扫描 `data/hanbing/` 目录下所有视频
2. **单视频片段池**：每个视频有自己的候选片段池
3. **最简单组合**：主视频 1-2 段 + 副视频 1 段
4. **基础兼容评分**：
   - 片段数量是否过多
   - 是否跨视频切太多次
   - 总长度是否太长
   - 文本主题是否明显跑偏
5. **接回总分**：兼容评分以 0.3 权重并回总分

## 使用方法

### 开启多视频模式

在 `settings.py` 中设置：

```python
BATCH_MULTI_VIDEO_ENABLE = True
```

### 运行

```bash
python batch_generator.py
```

当 `BATCH_MULTI_VIDEO_ENABLE = True` 且有 2 个以上视频时，自动走多视频流程。

否则按原来单视频流程处理。

## 评分逻辑

### 基础分

从每个片段的 `base_score` 取第一个有效值作为基础分。

### 兼容评分（满分 10 分）

初始 10 分，按以下维度扣分：

1. **片段数量过多**：超过 5 段，每多一段扣 0.5 分，最多扣 2 分
2. **跨视频切换过多**：
   - 切换 3 次以上：扣 1 分
   - 切换 2 次：扣 0.5 分
3. **总时长太长**：超过 60 秒，每 30 秒扣 1 分，最多扣 2 分
4. **文本连贯性**：后片段与前片段关键词不匹配，每次扣 0.2 分，最多扣 1.5 分

### 合并总分

```
总分 = 基础分 × 0.7 + 兼容评分 × 0.3
```

## 留白位置

当前版本不做：

- [ ] 多模态跨视频视觉匹配
- [ ] 复杂导演逻辑（自动选最佳切点）
- [ ] 三层以上模块拆分
- [ ] 任意多段自由组合
- [ ] 很重的主题理解

后续可扩展：

- [ ] 支持更多副视频（当前只用第一个）
- [ ] 支持自定义主副视频选择策略
- [ ] 增加视觉跨视频一致性评分
- [ ] 增加音频跨视频一致性评分
