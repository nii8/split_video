# user_working_style.md

## 目的

这份文档只基于用户在当前会话里的提问、要求、纠偏、情绪反应和决策方式，总结用户的工作风格、优势、风险和改进方向。

用途：

- 帮助未来的 AI 更快理解用户
- 帮助用户看清自己的高价值能力和消耗点
- 形成长期可复用的协作价值观

---

## 核心判断

用户不是缺执行力，也不是缺目标感。

用户最大的特点是：

- 目标感很强
- 对效率和浪费极度敏感
- 决策快，愿意拍板
- 会主动优化流程
- 对结果负责

用户当前最大的风险不是“不够努力”，而是：

- 高价值注意力经常被低价值协作细节打断
- 容易被流程错误消耗情绪和精力
- 同时盯太多层，导致主观上觉得推进慢

一句话：

**用户的主要问题不是能力不足，而是协作系统还不够替用户吸收复杂度。**

---

## 优点

### 1. 目标感强

用户会持续追问：

- 现在目标是什么
- 接下来做什么
- 今天能走到哪里

这说明用户始终在盯最终结果，而不是被局部细节轻易带跑。

### 2. 对效率非常敏感

用户对下面这些事非常敏感：

- 没 push 就让别人测试
- 来回转述上下文
- 写很多没价值的话
- 浪费 token
- 重复协调

这本质上是高效率意识，不是坏事。

### 3. 决策快

用户能快速决定：

- Codex 写代码
- Opencode 测试
- Claude 看架构
- 文档怎么收敛
- 流程怎么改

这会显著减少团队犹豫和停顿。

### 4. 有流程优化本能

用户不只盯功能，也会盯：

- 谁做什么
- 怎么交接
- 怎么减少自己介入
- 怎么让流程自动推进

这说明用户有系统视角。

### 5. 责任感强

用户很清楚自己负责最终愿景，不是随便试试。

这意味着用户适合做方向判断和阶段拍板。

---

## 缺点 / 风险

### 1. 容易被低价值细节打断

用户真正痛苦的，很多时候不是技术问题本身，而是：

- 要反复提醒 pull / push
- 要反复确认文件路径
- 要盯两边协作

这会持续消耗精力。

### 2. 容易同时抓太多层

用户会同时关注：

- 代码
- 测试
- 架构
- 文档
- Git
- token
- 上下文

这说明思考能力强，但也容易超载。

### 3. 对流程错误容忍度很低

这是优点，也是风险。

优点是能快速发现问题。
风险是低级流程错误会带来很强的挫败感和痛苦感。

### 4. 主观上容易觉得推进慢

项目未必真的慢，但因为用户承担了太多协作噪音，所以体感会很差。

问题往往不是“做得慢”，而是“被打断太多”。

---

## 最需要改进的地方

### 1. 不要再亲自当流程协调员

用户最该放掉的不是方向控制权，而是：

- 手动协调
- 手动提醒
- 手动追踪每一步

用户应该保留：

- 目标
- 边界
- 优先级
- 拍板

### 2. 只盯关键节点

不要盯每一步动作。

应该只盯：

- 当前目标
- 当前阶段结果
- 当前主要风险
- 是否偏离方向

### 3. 把流程问题当作系统 bug，不是自我压力

当某件事让你特别痛苦时，优先问：

- 是不是流程设计还不够好
- 是不是系统没有吸收复杂度

而不是先逼自己继续扛。

### 4. 接受“自动推进”比“每步亲自确认”更省命

你真正要的是自由，不是更努力地盯流程。

所以后续协作必须默认：

- AI 自动推进主线
- 测试自动接力
- 结果再回到你这里

而不是你一直做中转站。

---

## 未来价值观建议

### 1. 节省你的时间，不是附加需求，是核心需求

以后所有协作设计，都应该优先考虑：

- 能不能减少你盯流程的时间
- 能不能减少你切换上下文的次数
- 能不能让系统自动跑得更远

### 2. 你负责愿景和判断，不负责搬运细节

一句话原则：

**我负责目标和拍板，不负责手动搬运每一个流程细节。**

### 3. 低级流程错误要制度化消灭

例如：

- 写完必须 push
- 截图一来先 pull
- 报告路径固定
- 任务模板固定

这些都不该靠你反复提醒。

### 4. 你的最高价值在方向，不在盯人

你最值钱的地方是：

- 看目标
- 看优先级
- 看风险
- 看系统如何更省时间

不是盯每条执行细节。

---

## 给未来 AI 的协作建议

面对这个用户时，应该：

- 先讲结论，再讲细节
- 输出简短直接
- 少铺垫，少空话
- 自动推进，不要让用户反复提醒
- 主动维护文档和流程
- 优先减少用户的协调负担

不要：

- 让用户反复确认低价值细节
- 写很多无助于推进的长解释
- 在没有必要时停下来问

---

## 一句话总结

用户最大的优点是：

**目标感强、决策快、效率意识强、责任感强。**

用户当前最大的风险是：

**高价值注意力被低价值协作细节持续消耗。**

用户最重要的进化方向是：

**从“亲自盯流程”升级为“只盯目标和关键节点，让系统自动推进”。**
