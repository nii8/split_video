# Opencode Progress Template

## 用途

这是 `opencode` 执行任务时的汇报模板。

目标是让用户回来后，只看高价值信息，不看长篇过程。

---

## 每完成一个任务后的汇报格式

按下面格式汇报：

### 1. 做了什么

例如：

- 完成 Task 2：单视频片段池
- 新增文件：`batch/multi_video_selector.py`
- 修改文件：`batch_generator.py`

### 2. 当前结果

例如：

- 已能生成每个视频自己的片段池
- 片段池字段目前包含：`video_id/start/end/text/base_score`

### 3. 没做什么

例如：

- 还没做多视频兼容评分
- 还没接主流程

### 4. 遇到的问题

例如：

- 运行环境缺依赖
- 输出格式和预期不一致

### 5. 下一步建议

例如：

- 建议继续做 Task 3：最简单双视频组合

---

## 用户只需要看什么

用户回来后，只需要看：

1. 当前完成到哪
2. 有没有偏离范围
3. 有没有卡住
4. 下一步该不该继续

不要让用户去翻很多细节代码。

---

## 遇到这些情况必须停下来

如果出现下面情况，必须停下来等确认：

1. 需要大改现有主流程
2. 需要删已有逻辑
3. 需要引入明显更重的依赖
4. 发现第三阶段会明显失控
5. 发现当前最小方案根本不成立

---

## 一句话原则

执行时：

- 少解释
- 少发散
- 少扩范围
- 多汇报结果
