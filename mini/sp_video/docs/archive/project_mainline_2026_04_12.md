# 项目主线与行为轨迹

更新时间：2026-04-13（Asia/Shanghai）

## 当前主线

这个项目的主线已经进一步收口了：

- 最早是“长视频 + SRT -> 自动剪成短视频”的 CLI 工具
- 随后扩成给 OpenClaw / 机器人调用的 `skill.py` 流程
- 在 2026-03-29 之后，明显转向“批量预生成 + 自动评分 + 候选筛选”的系统
- 到 2026-04-13，当前真正的工作中心已经不是 `skill.py`，也不只是“打通多视频”，而是：

**把 `batch_generator.py` 推进成一套面向特定赛道的样本工厂：按时长分布批量出片、输出机器评分、承接人工回标、再持续改算法。**

## 按时间还原行为轨迹

### 2026-03-22

`TEST.md` 记录了 `skill.py` 的端到端测试结果。

这个阶段重点是把下面这套技能接口跑顺：

- 列视频
- 下载素材
- 生成脚本
- 匹配时间轴
- 生成视频并上传

说明当时项目的核心是给外部智能体调用。

### 2026-03-28

视频剪辑底层从旧方案转向 `ffmpeg filter_complex` 单次调用方案。

这是一次重要的执行链路简化，为后面的批量生成打基础。

### 2026-03-29

项目重心转向批量预生成系统。

核心思路变成：

- 大量生成候选
- 自动打分
- 只保留高质量结果

也就是从“单次交互生成”转向“自动候选工厂”。

### 2026-04-11 晚上

明确了 `batch_generator` V2 三阶段路线：

1. 视觉评分
2. 拼接自然度评分
3. 多视频组合生成

说明后续迭代已经不是零散加功能，而是在按明确路线推进。

### 2026-04-12

第三阶段“多视频组合最小骨架”接入实际输出链路。

这一步说明项目已经从“只输出候选和 summary”推进到了“可生成实际 mp4”。

### 2026-04-13

项目主线再次前进一步，重点不再只是“能不能生成视频”，而是：

- 是否能按时长分布稳定出样本
- 是否能让每个单视频源稳定生成固定数量
- 是否能让多视频输出满足时长门槛
- 是否能稳定输出机器评分表
- 是否能稳定输出人工评分模板
- 是否能形成“人工评分反推算法”的闭环

这说明项目已经开始从“功能打通期”切换到“数据闭环期”。

## 当前真实意图

当前最真实的意图不是继续打磨旧 CLI，也不是优先扩 `skill.py`，而是把 `batch_generator.py` 逐步升级成一个：

- 会批量生成不同长度分布的样本
- 会输出机器评分
- 会输出人工回标模板
- 会比较机器分与人工分偏差
- 会据此持续调整算法

也就是从“自动候选工厂”往“自动样本工厂 + 评分闭环系统”走。

## 当前最应该做的事

如果继续推进，优先级应当是：

1. 在真实运行环境验证单视频每源 `10` 条、多视频目标 `100` 条是否成立
2. 验证多视频最小时长门槛是否被真正满足
3. 验证机器评分表与人工评分模板是否能稳定产出
4. 完成一轮真实人工回标
5. 根据人工分与机器分的偏差，调整评分维度和权重

## 远期目标

远期目标不是简单“生成更多视频”，而是：

- 让系统能稳定批量出样本
- 让机器评分越来越接近人工偏好
- 让每轮测试都能产出可用于算法迭代的数据

## 终极目标

终极目标可以概括为：

**把项目从一套自动剪辑脚本，推进成面向特定赛道的自动短视频样本工厂与编排引擎。**

它最终要同时具备：

- 选题材
- 选时长
- 选片段
- 做编排
- 做机器初筛
- 承接人工反馈
- 根据反馈持续优化

## 未来三阶段路线

### 第一阶段：样本工厂跑稳

这一阶段先不追求“算法很聪明”，只追求：

- 单视频按目标数量稳定出样本
- 多视频按目标数量稳定出样本
- 时长分布基本符合配置
- 机器评分表稳定产出
- 人工评分模板稳定产出
- 测试报告能直接指出失败点

这一阶段的完成标准是：

**每一轮测试都能稳定产出一批可回标的视频样本。**

### 第二阶段：评分闭环跑稳

这一阶段的重点不再是“多生成一些”，而是：

- 完成多轮人工回标
- 找出高机器分低人工分的错判样本
- 找出低机器分高人工分的漏判样本
- 调整评分维度和权重
- 逐步让机器评分更接近人工偏好

这一阶段的完成标准是：

**机器评分开始具备实际筛选价值，而不是只有展示价值。**

### 第三阶段：赛道引擎化

这一阶段再往前走，才值得做更高级的东西：

- 赛道专属时长策略
- 赛道专属节奏模板
- 赛道专属编排策略
- 高分样本库沉淀
- 基于历史高分样本优化候选生成

这一阶段的完成标准是：

**系统不只是“会剪”，而是开始“懂这个赛道什么样的视频更可能高分”。**

## 当前统一目标表述

为了避免后续目标表达继续来回变化，当前统一使用下面这组表述：

### 终极目标

- 做成一个面向目标赛道的自动出片系统
- 不只负责生成视频，也负责承接机器评分、人工回标和算法迭代

### 近期目标

- 先把“批量出片 + 机器评分 + 人工回标 + 改算法”这条闭环跑顺
- 先让单视频和多视频都能稳定按目标数量和时长分布产出样本

### 当前重点

- 不继续发散扩功能
- 先把样本生成、评分、回标、迭代这条主线收口到稳定可复用

## 一句话结论

当前项目主线是：

`CLI 原型 -> skill.py 智能体接口 -> batch_generator 批量预生成系统 -> V2 三阶段增强 -> 当前进入“批量出样本 + 机器评分 + 人工回标 + 算法迭代”的闭环阶段`
