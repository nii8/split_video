## 问题 1: Phase1 超时具体卡点

### 卡点位置

**文件：** `batch/phase_runner.py::run_phase1_loop()`

**卡在第几行：** 第 15 行 `run_phase1_batch()` 调用

**具体表现：**
```python
for i in range(count):  # count = 20
    content = run_phase1_batch(video_id, srt_path, output_path)  # 单次 76-91 秒
```

### 实测数据

| 指标 | 数值 |
|------|------|
| 单次 AI 调用耗时 | 76-91 秒 |
| 默认调用次数 | 20 次 |
| Phase1 总耗时 | 约 1520-1820 秒（25-30 分钟） |
| 当前超时限制 | 120 秒（bash 工具超时） |

### 为什么会超时

**原因：** 默认配置 `BATCH_PHASE1_COUNT = 20`，意味着要调用 20 次 AI，每次 76-91 秒，总共需要 **25-30 分钟**。

但测试环境的 bash 超时限制是 120 秒（2 分钟），所以在第一次 AI 调用完成后就会因总超时被终止。

## 问题 2: 根因判断

### 根因分类

| 可能原因 | 是否成立 | 说明 |
|----------|---------|------|
| API Key 无效 | ❌ 不成立 | API Key 有效，调用成功 |
| 网络不通 | ❌ 不成立 | 网络连通正常 |
| 模型响应慢 | ✅ **成立** | 单次响应 76-91 秒 |
| 配置参数过重 | ✅ **成立** | 20 次 × 90 秒 = 30 分钟 |
| 代码逻辑错误 | ❌ 不成立 | 代码逻辑正确，能正常返回 |

### 核心根因

**AI 模型响应慢 + 批量参数过重**

1. **AI 响应慢**：`qwen3.5-plus` 模型单次响应需要 76-91 秒
   - 可能原因：prompt 较长（3000+ 字符）
   - 可能原因：模型本身推理慢
   - 可能原因：服务端排队

2. **批量参数过重**：默认配置不适合快速验证
   - Phase1: 20 次 × 90 秒 = 30 分钟
   - Phase2: 100 次 × 90 秒 = 150 分钟
   - 总计：**超过 3 小时**

### 为什么之前单元测试能通过

之前测试使用的是 **mock 数据**（直接读取已有的 phase3 结果），绕过了 AI 调用环节。

一旦进入真实流程，AI 调用耗时就暴露了。

## 问题 4: 默认参数是否过重

### 当前配置

```python
# settings.py
BATCH_PHASE1_COUNT = 20    # 20 次字幕筛选
BATCH_PHASE2_COUNT = 100   # 100 次脚本生成
```

### 实际耗时估算

| 阶段 | 次数 | 单次耗时 | 总耗时 |
|------|------|----------|--------|
| Phase1 | 20 | 90 秒 | 30 分钟 |
| Phase2 | 100 | 90 秒 | 150 分钟 |
| Phase3 | - | <1 秒 | <1 分钟 |
| **总计** | - | - | **约 3 小时** |

### 结论

**✅ 是的，当前参数明显过重**

设计意图：
- 20 次 Phase1 是为了筛选多个不同角度
- 100 次 Phase2 是为了生成大量候选脚本

适用场景：
- 生产环境批量预生成（可以跑几小时）
- 离线任务

不适用场景：
- 快速验证和测试
- 实时交互
- 开发调试

## 下一步建议

### 立即可做（配置调整）

1. **添加测试模式开关**
   ```python
   # settings.py
   BATCH_TEST_MODE = False  # 测试模式：Phase1=1, Phase2=1
   
   if BATCH_TEST_MODE:
       BATCH_PHASE1_COUNT = 1
       BATCH_PHASE2_COUNT = 1
   ```

2. **或在 `process_multi_video()` 中临时降低参数**
   ```python
   # 多视频模式下自动降低参数
   old_phase1_count = settings.BATCH_PHASE1_COUNT
   old_phase2_count = settings.BATCH_PHASE2_COUNT
   settings.BATCH_PHASE1_COUNT = 1
   settings.BATCH_PHASE2_COUNT = 1
   
   # ... 运行 ...
   
   # 恢复原配置
   settings.BATCH_PHASE1_COUNT = old_phase1_count
   settings.BATCH_PHASE2_COUNT = old_phase2_count
   ```

### 建议评估（架构层面）

1. **评估是否更换 AI 模型**
   - 当前：`qwen3.5-plus`（76-91 秒/次）
   - 备选：`qwen3.5` 或其他更快模型
   - 权衡：速度 vs 质量

2. **评估批量参数设计**
   - 当前设计意图：大量候选，优中选优
   - 问题：测试验证成本过高
   - 建议：分场景配置（测试 vs 生产）

3. **增加超时和重试机制**
   - 单次 AI 调用超时：建议 120 秒
   - 失败重试：建议 2-3 次
   - 失败降级：使用缓存或跳过

### 测试建议

   - 当前 `data/batch_results/` 下已有 Phase1-3 结果
   - 可直接用于验证多视频候选生成
   - 无需每次都跑完整 AI 流程
